## 二、AI Coding

### 1. 最近动态

#### 🌏 海外

- 🔴 **[OpenAI Codex CLI 0.107.0 发布：线程分叉多 Agent](https://developers.openai.com/codex/changelog/)** - OpenAI · 2026-03-02
  > 新版 Codex CLI 支持将对话线程分叉为子 Agent 并行工作，自定义工具可返回多模态输出（含图片），记忆系统新增配置和重置命令。多 Agent 编排能力大幅增强。

- 🔴 **[Latent Space: "How to Kill the Code Review"](https://latent.space/p/reviews-dead)** - Ankit Jain / Latent Space · 2026-03-03（今日）
  > Aviator CEO Ankit Jain 在 Latent Space 发表长文，主张"人工写的代码已死于 2025，代码审查将死于 2026"。提出 Spec-driven Development：审查规格说明而非代码，让 AI 从规格生成代码。

- **[GPT-5.3-Codex-Spark：1000+ token/s 实时编码](https://developers.openai.com/codex/changelog/)** - OpenAI · 2026-02-12
  > OpenAI 发布首个实时编码模型 Codex-Spark，基于 Cerebras 合作实现 1000+ token/s 推理速度。128k 上下文窗口，Pro 用户可用。标志着 AI 编码从"等待生成"进入"实时交互"阶段。

- **[ChatGPT 新增交互式代码块](https://help.openai.com/en/articles/6825453-chatgpt-release-notes)** - OpenAI · 2026-02-19
  > ChatGPT 内可直接编写、编辑和预览代码，支持分屏视图和图表/迷你应用预览，进一步模糊 IDE 与聊天机器人的界限。

#### 🇨🇳 国内

- **[智谱 GLM-5 主打编程 Agent 场景](https://www.eet-china.com/mp/a475977.html)** - 智谱AI · 2026-02-12
  > GLM-5 定位为"编程与智能体"专用模型，擅长复杂系统工程和长程 Agent 任务。国内模型在 AI Coding 赛道的差异化竞争正在形成。

- **[中国 AI 模型 API 调用量首超美国](https://vip.stock.finance.sina.com.cn/)** - 中国银河证券 · 2026-03-02
  > 数字经济双周报显示，中国 AI 模型应用量首次超过美国。AI Coding 是增长最快的应用场景之一。

### 2. 热门观点

> **Ankit Jain** · Aviator CEO / Latent Space 特约
> "Human-written code died in 2025. Code reviews will die in 2026. Specs become the source of truth. Code becomes an artifact. You don't review artifacts — you review specs."
> *—— [Latent Space](https://latent.space/p/reviews-dead) · 2026-03-03*

> **Gergely Orosz** · Pragmatic Engineer
> "The latest AI tools have cleared my own quality bar of 'code better than the median developer'... We're approaching the point where AI writes almost all code."
> *—— [Pragmatic Engineer](https://newsletter.pragmaticengineer.com/p/when-ai-writes-almost-all-code-what) · 2026-01-06*

### 3. 趋势洞察

AI 编程工具的竞争已从"单 Agent 辅助"进入**"多 Agent 编排"**时代。Codex 0.107 的线程分叉、Codex-Spark 的实时编码、ChatGPT 的交互式代码块——三条产品线同时进化，OpenAI 正在构建从"实时对话"到"后台并行"的全栈 AI 编程体验。而 Latent Space 今日文章标志着行业认知转折点："审查代码"让位于"审查规格"。

**对从业者的建议**：开始实验 Spec-driven 工作流——先写清楚"要什么"（规格），再让 AI 生成代码和测试。短期不必激进替换，但应在小项目上试跑、积累经验。关注 Codex 的多 Agent 能力。

**值得警惕**：Pragmatic Engineer 虽然肯定了 AI 编码质量，但也指出"AI 写几乎所有代码"之后，**工程师的角色变化并不意味着工程师变得不重要**——理解系统、定义规格、把控质量的能力可能更加稀缺。

### 3. 趋势洞察
